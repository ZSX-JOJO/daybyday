列表

列表--无序列表
  • ul unordered list 无序列表
  • li list item 列表项
  • 此为组标签，要么不写，写就要写一组
  • li不能单独存在，必须包裹在ul或者ol中
  • ul或者ol的儿子只可以是li
  • li是一个 容器级 标签，li里面什么都能放,可以继续放置ul或者ol
列表--有序列表
  1. ol ordered list 有序列表
  2. 其他与ul类似
列表--定义列表
dl definition list 定义列表
dt definition title 定义标题
dd definition description 定义表述词 dd 用来描述dt
dt、dd只能在dl里面；dl里面只能有dt、dd
一个dt可以配置很多dd
dt、dd都是容器级标签，想放什么都可以

div和span

div的语义是division“分割”； span的语义就是span“范围、跨度”
div标签是一个容器级标签，里面什么都能放，甚至可以放div自己
span也是表达“小区域、小跨度”的标签，但是是一个“文本级”的标签
span里面只能放置文字、图片、表单元素。 span里面不能放p、h、ul、dl、ol、div

表单

form表单
action属性就是表示，表单将提交到哪里。 method属性表示用什么HTTP方法提交，有get、post两种
文本框
input type="text"
密码框
input type="password"
单选按钮
input type="radio" name="***"
单选按钮，天生是不能互斥的，如果想互斥，必须要有相同的name属性。name就是“名字”
默认被选择，就应该书写checked=”checked” 或者
复选框
input标签 type是checkbox
input type="checkbox" name="***"
复选框，最好也是有相同的name（虽然他不需要互斥，但是也要有相同的name）
下拉列表
select就是“选择”，option“选项”,select标签和ul、ol、dl一样，都是组标签
多行文本框(文本域)
rows 行 columns 列
三种按钮
<input type="button" value="我是一个普通按钮" />普通按钮
<input type="submit" />提交按钮
<input type="reset" />重置按钮
label标签
input元素要有一个id，然后label标签就有一个for属性，和id相同，就表示绑定了，这个label和input就有绑定关系了
    <input type="radio" name="sex"id="nan" /><label for="nan">男</label>
    <input type="radio" name="sex"id="nv" /><label for="nv">女</label>
     
    <input type="checkbox" id="kk" /><label for="kk">10天内免登陆</label>
    <input type="checkbox" id="kk2" /><label for="kk2">10天内免登陆2</label>
所有表单元素都有label

css高级选择器

后代选择器 (空格表示后代)
.div1 p{ color:red; }
交集选择器 (用.号表示)
h3.special{ color:red;3 }
并集选择器(分组选择器 使用,号表示)
h3,li{ color:red; }
通配符*
*{ color:red; }
儿子选择器(用>表示,区别于后代选择器)
div>p{ color:red; }
序选择器
ul li:first-child{ color:red; }
下一个兄弟选择器+
h3+p{ color:red; }
